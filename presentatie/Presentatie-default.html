Fetch API: Data Ophalen

De moderne manier om data op te halen:

async function getProducts() { const response = await fetch('https://api.example.com/products'); const data = await response.json(); console.log(data); return data; }

Stap 1: fetch() haalt de response op

Stap 2: .json() zet het om naar een JavaScript object

CREATE: POST Request

async function createProduct(name, price) { const response = await fetch('/api/products', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ name: name, price: price }) }); const result = await response.json(); console.log('Aangemaakt!', result); }

method: 'POST' voor aanmaken

headers: Vertel de server dat we JSON sturen

body: De data als JSON string

UPDATE: PUT Request

async function updateProduct(id, name, price) { const response = await fetch(`/api/products/${id}`, { method: 'PUT', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ id: id, name: name, price: price }) }); if (response.ok) { console.log('Product ge√ºpdatet!'); } }
Let op: De ID zit vaak in de URL: /api/products/42

DELETE: Verwijderen

async function deleteProduct(id) { // Vraag bevestiging aan gebruiker const confirmed = confirm('Weet je het zeker?'); if (!confirmed) return; const response = await fetch(`/api/products/${id}`, { method: 'DELETE' }); if (response.ok) { console.log('Verwijderd!'); loadProducts(); // Ververs de lijst } }
Tip: Vraag altijd bevestiging bij delete acties!

Error Handling

async function fetchData(url) { try { const response = await fetch(url); // Check HTTP status if (!response.ok) { throw new Error(`HTTP Error: ${response.status}`); } return await response.json(); } catch (error) { console.error('Fout:', error); showMessage('Er ging iets mis!'); } }

Let op: fetch() gooit GEEN error bij 404 of 500!

Check altijd response.ok om fouten te detecteren!

HTTP Status Codes

Code Betekenis Wanneer?

response.ok is true bij status 200-299

Best Practices

Do's ‚úÖ

  • Gebruik async/await
  • Check response.ok
  • Toon loading states
  • Valideer input
  • Bevestig delete acties

Don'ts ‚ùå

  • Vergeet geen try/catch
  • Geen wachtwoorden in URL
  • Negeer geen HTTP errors
  • Vergeet geen preventDefault()
  • Geen synchrone requests

Samenvatting

  • AJAX = data uitwisselen zonder page refresh
  • CRUD = Create, Read, Update, Delete
  • fetch() voor alle HTTP requests
  • GET = ophalen, POST = aanmaken
  • PUT = updaten, DELETE = verwijderen
  • Altijd response.ok checken!
  • Gebruik try/catch voor errors
  • JSON.stringify() voor verzenden, .json() voor ontvangen
